- id: P0-17
  title: "Fix export_tables_for_translation.ts — writes to directory not file"
  file: scripts/export_tables_for_translation.ts
  line: 25
  root_cause: >
    Line 25 of scripts/export_tables_for_translation.ts sets:
    `const filePath = path.resolve(process.cwd(), "locales", "content")`
    This resolves to the locales/content/ directory path, not a file path.
    Line 28 then passes this directory path to `fs.writeFileSync(filePath, ...)`,
    which throws `EISDIR: illegal operation on a directory` when
    locales/content/ exists, or creates a file named "content" in the locales/
    directory when it does not. The commented-out line above (`// const filePath
    = 'app/locales/content/en.json'`) shows the intended destination. The fix
    is to append the filename so the path resolves to locales/content/en.json.
  fix_complexity: Trivial
  requires_spec: false
  requires_test: false
  fix_summary: >
    Change the filePath assignment at line 25 to
    path.resolve(process.cwd(), "locales", "content", "en.json") so
    writeFileSync writes to a file rather than a directory.
  antipattern: null
