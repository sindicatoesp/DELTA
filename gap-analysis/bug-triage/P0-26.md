---
- id: P0-26
  title: "Create SECURITY.md — vulnerability disclosure policy and contact"
  file: null
  line: null
  root_cause: >
    SECURITY.md does not exist in the repository. Without it, security
    researchers have no guidance on how to responsibly disclose vulnerabilities
    and may resort to public issue creation, exposing production deployments
    before a fix is available. The item notes require the file to state: which
    versions receive security fixes, how to report a vulnerability via a private
    channel (not the public issue tracker), the expected response SLA, and
    whether a coordinated disclosure or bug bounty programme exists.
  fix_complexity: Simple
  requires_spec: false
  requires_test: false
  fix_summary: >
    Create SECURITY.md covering supported versions, private disclosure
    instructions (e.g. email to a security@ address or GitHub private
    vulnerability reporting), response SLA, and disclosure policy; explicitly
    state that no bug bounty programme currently exists if that is the case.
  antipattern: null
