---
- id: P0-20
  title: "Add CSP header to entry.server.tsx — missing from production"
  file: app/entry.server.tsx
  line: 86
  root_cause: >
    entry.server.tsx sets several security headers on all responses
    (Referrer-Policy, Permissions-Policy, X-Frame-Options, X-XSS-Protection,
    X-Content-Type-Options, Cache-Control) in both handleBotRequest (line 86)
    and handleBrowserRequest, but no Content-Security-Policy header is set.
    Without CSP, inline scripts and cross-origin resource loading are unrestricted,
    leaving XSS payloads free to execute. The app uses React Router v7 SSR,
    PrimeReact, and Vite-built assets, which require a carefully constructed
    CSP that allows self-hosted scripts and the app's CDN/font origins.
  fix_complexity: Complex
  requires_spec: true
  requires_test: true
  fix_summary: >
    Audit all script, style, font, image, and connect origins used by the app,
    then add a Content-Security-Policy header in both handleBotRequest and
    handleBrowserRequest in entry.server.tsx; start with report-only mode to
    capture violations before enforcing.
  antipattern: null
