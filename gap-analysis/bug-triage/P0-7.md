- id: P0-7
  title: "Fix deleteAllData silent error swallow"
  file: app/backend.server/handlers/human_effects.ts
  line: 371
  root_cause: >
    The deleteAllData function (line 371) iterates over human-effects table
    definitions and calls clear() for each, then checks `if (!r.ok) { return r; }`.
    The clear() function returns Response.json({ok: false, error: ...}) on error,
    which creates an HTTP Response with status 200. The check `r.ok` evaluates
    the HTTP Response.ok property (true for status 200-299), not the JSON body
    field. Because clear() always returns a 200 status even for errors, r.ok is
    always true and the error branch never executes. Errors from individual table
    clears are silently swallowed and the loop continues, leaving data in a
    partially deleted state.
  fix_complexity: Simple
  requires_spec: false
  requires_test: true
  fix_summary: >
    Replace the HTTP Response pattern in clear() with a typed result object
    { ok: boolean, error?: ... } so that deleteAllData can correctly check the
    application-level ok field rather than the HTTP status ok property.
  antipattern: AP-12
