- id: P0-9
  title: "Fix handleTransaction sentinel string"
  file: app/backend.server/models/common.ts
  line: 58
  root_cause: >
    Line 58 declares `export const TransactionAbortError = "TransactionAbortError"`
    — a plain string constant used as a sentinel. handleTransaction (line 64)
    throws this string at line 78 (`throw TransactionAbortError`) and catches it
    with a string-equality check at line 80 (`if (error !== TransactionAbortError)`).
    TypeScript cannot perform control-flow narrowing on string throws, so callers
    that catch errors lose type safety. Any unrelated string thrown inside a
    transaction would be caught and silently swallowed by the sentinel check.
    The pattern also makes it impossible to attach a stack trace to the abort
    signal. handleCreateTransaction and handleUpdateTransaction both call this
    function (lines 87, 93).
  fix_complexity: Simple
  requires_spec: false
  requires_test: true
  fix_summary: >
    Replace the string constant with a class TransactionAbortError extends Error {},
    throw new TransactionAbortError() at line 78, and check
    `if (!(error instanceof TransactionAbortError))` at line 80.
  antipattern: AP-06
