- id: P0-14
  title: "Fix sanitizeInput — remove destructive quote stripping"
  file: app/utils/security.ts
  line: 17
  root_cause: >
    sanitizeInput() in app/utils/security.ts applies a regex at line 17 that
    strips the characters `'`, `"`, and `;` from all input:
    `input = input.replace(/['";]/g, "")`. Stripping apostrophes corrupts
    French names (e.g. "L'Aquila" → "LAquila"), Arabic possessives, Irish
    surnames (O'Brien → OBrien), and any user-generated text containing
    apostrophes. Stripping double-quotes also removes legitimate textual content.
    The function is named sanitizeInput and is intended for XSS/SQL injection
    prevention, but parametrised Drizzle queries already prevent SQL injection;
    the apostrophe removal provides no security benefit and causes data corruption.
  fix_complexity: Simple
  requires_spec: false
  requires_test: true
  fix_summary: >
    Remove the apostrophe character from the regex in sanitizeInput so that
    only angle-bracket HTML tags and semicolons are stripped; review all call
    sites to determine whether even those removals are necessary given Drizzle's
    parameterised queries.
  antipattern: AP-11
