---
- id: P0-5
  title: "Fix placeholder support email in ErrorBoundary"
  file: app/root.tsx
  line: 326
  root_cause: >
    app/root.tsx contains two hardcoded occurrences of support@example.org:
    line 326 as an href="mailto:support@example.org" and line 329 as the visible
    link text. This placeholder email is displayed in the ErrorBoundary component
    shown to users when an unhandled error occurs. In production every deployment
    shows a non-functional support link, making it impossible for users to contact
    the correct support team. The correct fix is to read the address from a
    SUPPORT_EMAIL environment variable exposed to the client via the root loader.
  fix_complexity: Trivial
  requires_spec: false
  requires_test: false
  fix_summary: >
    Replace the two hardcoded support@example.org strings with the value of a
    SUPPORT_EMAIL env var (with a sensible fallback); add SUPPORT_EMAIL to
    example.env with a comment.
  antipattern: null
