  title: "Fix type export bugs in humanCategoryPresence + hipHazard"
  file: app/drizzle/schema/hipHazardTable.ts

  root_cause: >
    app/drizzle/schema/hipHazardTable.ts exports two types at lines 32-33 with
    incorrect names and wrong source tables. Line 32 reads
    `export type SelectDisasterRecords = typeof disasterRecordsTable.$inferSelect`
    and line 33 reads
    `export type InsertDisasterRecords = typeof disasterRecordsTable.$inferInsert`.
    Both names say "DisasterRecords" and both infer from disasterRecordsTable
    instead of hipHazardTable — a copy-paste error. Any code importing
    SelectDisasterRecords or InsertDisasterRecords from hipHazardTable.ts
    receives the disaster records schema instead of the hip hazard schema.
    The humanCategoryPresenceTable.ts types are correctly named and are not
    affected.
  fix_complexity: Trivial
  requires_spec: false
  requires_test: false
  fix_summary: >
    Rename the two exported types in hipHazardTable.ts to SelectHipHazard and
    InsertHipHazard, and change the right-hand side to reference hipHazardTable
    instead of disasterRecordsTable.
